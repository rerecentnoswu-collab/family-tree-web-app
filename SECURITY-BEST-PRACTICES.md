# Security Best Practices for Genealogy App

## 🔴 CRITICAL SECURITY ISSUES IN CURRENT VERSION

### 1. No Authentication ⚠️ CRITICAL
**Current State:** Anyone can access all family data
**Risk:** Complete data breach, unauthorized access, data manipulation

**Solution:**
```typescript
// Enable Supabase Auth
import { supabase } from './utils/supabase/client-secure';

// Sign up
await supabase.auth.signUp({ email, password });

// Sign in
await supabase.auth.signInWithPassword({ email, password });
```

### 2. Hardcoded API Keys ⚠️ SEVERE
**Current State:** API keys in `/utils/supabase/info.tsx`
**Risk:** Anyone with code access can use your Supabase

**Solution:**
```bash
# .env.local (NEVER commit this file!)
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your_anon_key_here

# Add to .gitignore
echo ".env.local" >> .gitignore
echo "utils/supabase/info.tsx" >> .gitignore
```

### 3. Public Database Access ⚠️ CRITICAL
**Current State:** RLS policies allow public read/write
**Risk:** Anyone can create, read, update, or delete ALL records

**Solution:** Run `SECURITY-IMPROVEMENTS.sql` instead of `supabase-setup.sql`

#### Step 2: Enable Authentication
```typescript
// 1. Replace client.ts with client-secure.ts
mv utils/supabase/client-secure.ts utils/supabase/client.ts

// 2. Create AuthProvider.tsx
import { createContext, useContext, useEffect, useState } from 'react';
import { supabase } from '/utils/supabase/client';

const AuthContext = createContext({});

export function AuthProvider({ children }) {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    // Check active session
    supabase.auth.getSession().then(({ data: { session } }) => {
      setUser(session?.user ?? null);
      setLoading(false);
    });

    // Listen for auth changes
    const { data: { subscription } } = supabase.auth.onAuthStateChange(
      (_event, session) => {
        setUser(session?.user ?? null);
      }
    );

    return () => subscription.unsubscribe();
  }, []);

  return (
    <AuthContext.Provider value={{ user, loading }}>
      {children}
    </AuthContext.Provider>
  );
}

export const useAuth = () => useContext(AuthContext);
```

#### Rate Limiting (Server-Side)
```sql
-- Add to Supabase Functions
CREATE OR REPLACE FUNCTION check_rate_limit(
  user_id UUID,
  action TEXT,
  max_requests INTEGER,
  window_seconds INTEGER
)
RETURNS BOOLEAN AS $$
DECLARE
  request_count INTEGER;
BEGIN
  SELECT COUNT(*) INTO request_count
  FROM rate_limits
  WHERE user_id = user_id
    AND action = action
    AND window_start > NOW() - (window_seconds || ' seconds')::INTERVAL;

  IF request_count >= max_requests THEN
    RETURN FALSE;
  END IF;

  INSERT INTO rate_limits (user_id, action)
  VALUES (user_id, action);

  RETURN TRUE;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;
```

```typescript
// Encrypt sensitive fields before storage
import CryptoJS from 'crypto-js';

function encryptField(value: string, key: string): string {
  return CryptoJS.AES.encrypt(value, key).toString();
}

function decryptField(encrypted: string, key: string): string {
  const bytes = CryptoJS.AES.decrypt(encrypted, key);
  return bytes.toString(CryptoJS.enc.Utf8);
}
```

## 🚨 INCIDENT RESPONSE PLAN

### If API Keys Are Compromised:
1. **Immediately rotate keys** in Supabase Dashboard
2. **Revoke all sessions**: `supabase auth sessions revoke`
3. **Review audit logs** for suspicious activity
4. **Change all affected passwords**
5. **Notify users** if data was accessed

### If Unauthorized Access Detected:
1. **Isolate affected accounts**
2. **Review RLS policies** for gaps
3. **Check audit logs** for entry point
4. **Restore from backup** if data corrupted
5. **Patch vulnerability**
6. **Document incident** for future reference

## 🧪 SECURITY TESTING

### Manual Testing:
```bash
# Test RLS policies
psql -c "SELECT * FROM persons" # Should fail without auth

# Test SQL injection
curl -X POST https://your-api.com/persons \
  -d "first_name='; DROP TABLE persons;--"  # Should be blocked

# Test XSS
# Try entering: <script>alert('XSS')</script> in form fields
```

### Automated Testing:
```typescript
// security.test.ts
describe('Security Tests', () => {
  it('should require authentication', async () => {
    const { error } = await supabase.from('persons').select();
    expect(error).toBeTruthy();
  });

  it('should prevent SQL injection', async () => {
    const malicious = "'; DROP TABLE persons;--";
    await expect(addPerson({ first_name: malicious })).rejects.toThrow();
  });

  it('should sanitize XSS', () => {
    const xss = '<script>alert("XSS")</script>';
    const sanitized = sanitizeString(xss);
    expect(sanitized).not.toContain('<script>');
  });
});
```
